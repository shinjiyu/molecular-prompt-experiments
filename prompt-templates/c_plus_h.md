# 组合模板（C+H - 共价键+氢键）

## 核心思想

将共价键（强逻辑）和氢键（反思验证）结合，形成"推进-检查"循环模式：
1. 构建几步逻辑链（推进）
2. 对这些步骤进行反思验证（检查）
3. 决策：继续推进或回退修正

## 模板内容

```
【目标】构建严密的逻辑链，并在关键节点进行反思验证

采用"推进-检查"循环模式：

【Phase 1: 推进】（构建2-3步逻辑链）
→ 步骤N: [基于前序步骤的逻辑推理]
→ 步骤N+1: ...
→ 步骤N+2: ...

【Phase 2: 检查】（对上述步骤进行反思）
→ 检查点:
   ├─ 逻辑连贯性: [步骤间是否合理衔接]
   ├─ 前提可靠性: [是否依赖未经证实的前提]
   └─ 结论合理性: [中间结论是否合理]

【Phase 3: 决策】
→ 如果检查通过 → 继续 Phase 1
→ 如果发现问题 → 回退并修正

循环直到得到最终答案。

【格式示例】
[推进1] 因为A，所以B
[推进2] 因为B，所以C
[检查1] 验证 B→C 的逻辑
   └─ 结论: 通过
[推进3] 因为C，所以D
[检查2] 验证 C→D 的逻辑
   └─ 结论: 发现问题 - C的前提不完整
[回退] 回到推进2，重新分析...

现在请解决以下问题：
{question}
```

## 适用场景

- ✅ 复杂数学证明
- ✅ 多步骤逻辑推理
- ✅ 代码调试（需要逐步验证）
- ✅ 高可靠性要求的场景

## 不适用场景

- ❌ 简单直接的问题
- ❌ 需要快速响应的场景
- ❌ 创意探索类问题

## 预期效果

- **优势**：兼顾严密性和可靠性
- **劣势**：推理速度慢，Token 消耗大
- **键分布**：共价键 ~60%，氢键 ~20%

## 使用建议

1. 在关键决策点进行检查
2. 不要每步都检查（效率问题）
3. 发现问题时及时回退

## 推进-检查循环示例

```
问题：证明 n³ - n 能被6整除

[推进1] n³ - n = n(n² - 1) = n(n-1)(n+1)
[推进2] 这是三个连续整数的乘积
[检查1] 验证：连续整数乘积的特性
   ├─ 逻辑连贯性: ✅
   ├─ 前提可靠性: ✅
   └─ 结论合理性: ✅
   └─ 结论: 通过

[推进3] 三个连续整数中必有一个是3的倍数
[推进4] 三个连续整数中必有偶数
[检查2] 验证：3的倍数和偶数的存在性
   ├─ 逻辑连贯性: ✅
   ├─ 前提可靠性: ✅
   └─ 结论合理性: ✅
   └─ 结论: 通过

[推进5] 因此 n(n-1)(n+1) 能被6整除
[最终结论] n³ - n 能被6整除，证明完成
```
