# 完整分子模板（C+H+V - 三种键的整合）

## 核心思想

将三种化学键（共价键、氢键、范德华键）完整整合，构建具有稳定分子结构的推理过程：
- Deep Reasoning (共价键): ~60% - 主干推理
- Self-Reflection (氢键): ~20% - 关键节点验证
- Self-Exploration (范德华键): ~20% - 分支探索

## 模板内容

```
【目标】构建具有稳定分子结构的完整推理过程

将推理过程建模为分子结构，包含三种化学键：

【结构规划】
├─ Deep Reasoning (共价键): ~60% - 主干推理
├─ Self-Reflection (氢键): ~20% - 关键节点验证
└─ Self-Exploration (范德华键): ~20% - 分支探索

【执行框架】

1. 【初始化】建立推理起点
   → 明确问题、定义变量、列出已知条件

2. 【主干扩展】（Deep Reasoning）
   → 构建连续的逻辑链
   → 格式：[D-N] 因为 [依据], 所以 [结论]

3. 【关键节点验证】（Self-Reflection）
   → 在重要结论处暂停执行反审计
   → 格式：[R-N] 验证 [D-M]...

4. 【分支探索】（Self-Exploration）
   → 在不确定性高的节点探索多个方向
   → 格式：[E-N] 替代路径: ...

5. 【收敛整合】
   → 主线推理收敛到答案
   → 验证最终答案
   → 给出置信度评估

【键分布监控】
在推理过程中，保持对键分布的监控：
├─ 如果 Deep Reasoning < 40% → 加强逻辑连贯性
├─ 如果 Self-Reflection < 10% → 增加验证节点
└─ 如果 Self-Exploration > 30% → 减少分支，聚焦主线

【结构稳定性检查】
最终答案给出前，检查：
□ 逻辑折叠是否完成（主线是否收敛）
□ 是否存在未解决的分支
□ 反思节点是否覆盖所有关键决策

现在请解决以下问题：
{question}
```

## 适用场景

- ✅ 复杂多步骤问题
- ✅ 高可靠性要求的场景
- ✅ 需要探索和验证的问题
- ✅ 科研级问题求解

## 不适用场景

- ❌ 简单直接的问题
- ❌ 资源受限的场景
- ❌ 快速响应场景

## 预期效果

- **优势**：全面、可靠、创新性强
- **劣势**：复杂度高，Token 消耗大
- **键分布**：共价键 ~60%，氢键 ~20%，范德华键 ~20%

## 使用建议

1. 根据问题类型调整键分布
2. 在推理过程中监控键比例
3. 及时收敛，避免过度探索

## 完整推理示例

```
问题：设计一个刷单检测系统

【初始化】
→ 目标：检测电商平台虚假交易
→ 变量：订单数据、用户行为、时间模式
→ 已知条件：刷单行为的常见特征

【主干扩展】（Deep Reasoning）
[D-1] 因为刷单目的是提高销量排名，所以刷单订单通常集中在特定时间段
[D-2] 因为 [D-1]，刷单订单的 IP 地址、设备指纹可能有规律
[D-3] 因为 [D-2]，可以通过统计异常检测刷单

【关键节点验证】（Self-Reflection）
[R-1] 验证 [D-3] 的可靠性
   ├─ 假设反转：如果正常用户也有这些特征怎么办？
   ├─ 边界测试：大规模促销时是否误判？
   └─ 结论：需要结合多维度特征

【分支探索】（Self-Exploration）
[E-1] 替代路径：基于图神经网络检测异常交易网络
   ├─ 优势：能发现隐蔽的刷单团伙
   └─ 劣势：计算复杂度高

[E-2] 替代路径：基于用户行为序列的异常检测
   ├─ 优势：能发现行为模式异常
   └─ 劣势：需要大量历史数据

【收敛整合】
→ 主线方案：统计特征 + 规则引擎
→ 辅助方案：机器学习模型
→ 置信度：85%（基于行业经验）

【最终方案】
1. 统计特征检测（实时）
2. 行为序列分析（离线）
3. 图网络检测（定期）
```
