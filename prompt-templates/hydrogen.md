# 氢键模板（Self-Reflection - 反思验证）

## 核心思想

氢键代表自我反思和验证："后续步骤验证/修正前期步骤"。这种推理模式强调：
- 在每个关键步骤后主动寻找错误
- 建立负反馈回路
- 通过验证提高推理可靠性

## 模板内容

```
【目标】在每个关键步骤后主动寻找错误

采用"怀疑一切"的思维方式：

每完成一个推理步骤后，立即进行"反审计"：
1. 【假设反转】如果这个结论是错的，可能因为什么？
2. 【边界测试】这个结论在什么情况下不成立？
3. 【量级检查】结果的数量级是否合理？

【格式】
推理步骤: [内容]
├─ 反审计 1: [假设反转] ...
├─ 反审计 2: [边界测试] ...
└─ 反审计 3: [量级检查] ...
   └─ 结论: [继续/修正/回退]

只有当三个反审计都通过时，才能进入下一步。

现在请解决以下问题：
{question}
```

## 适用场景

- ✅ 复杂数学计算（需要检查中间结果）
- ✅ 逻辑推理（需要验证假设）
- ✅ 代码调试（需要确认修复正确）
- ✅ 高风险决策问题

## 不适用场景

- ❌ 简单直接的问题
- ❌ 需要快速响应的场景
- ❌ 创意探索类问题

## 预期效果

- **优势**：错误率低，推理可靠性高
- **劣势**：推理速度慢，Token 消耗大
- **键分布**：氢键占比 15-25%

## 使用建议

1. 在关键决策点使用反思
2. 不要过度反思简单步骤（效率问题）
3. 与共价键结合使用效果最佳

## 反审计检查清单

- [ ] 假设是否合理？
- [ ] 逻辑链条是否完整？
- [ ] 数量级是否合理？
- [ ] 是否存在边界情况？
- [ ] 是否有遗漏的前提条件？
